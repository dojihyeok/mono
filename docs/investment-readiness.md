# MONO MVP 투자/우승 적합성 진단 (Investment & Competition Readiness)

> 작성: 2026-06-23 · 목적: 현재 MVP가 **모두의창업 우승** 및 **VC/SI 투자 유치**에 필요한 기능·근거를 갖췄는지 냉정하게 점검.
> v1 = 직접 진단(코드·문서 정독 기반). 5-렌즈 평가 워크플로우 결과는 완료 시 §6에 추가.

---

## 0. 한 줄 결론

**실행력·제품 완성도는 대회 상위권급이지만, 투자·우승의 결정타인 "실제 검증 데이터"와 "공급(일자리) 확보"가 비어 있다.** 지금은 *잘 만든 데모*, 필요한 건 *작동하는 증거*. → **빠진 건 기능이 아니라, 기능 위에 쌓일 트랙션·공급·수익·PoC.**

---

## 1. 모두의창업 우승 — 부분 준비 (≈65/100)

| 강점 | 갭(우승에 치명적) |
|---|---|
| 5개 화면(사용자앱·기업웹·관리자·분석웹) 端-to-端 작동 — MVP치고 드문 완성도 | **실데이터 0**: 분석 대시보드(퍼널·리텐션·Aha)는 골격만, 채울 숫자가 없음 → "스토리만 있고 증거 없음" |
| Demand-first 루프 닫힘(공고→승인→노출→지원→배정→출역) = 핵심 가설 시연 가능 | **차별·해자 미입증**: coworkers·출역 데이터 해자는 *주장*일 뿐 데이터로 안 보임 |
| 이벤트 로그·퍼널·리텐션 측정 인프라 = 심사가 좋아하는 "검증형" 태세 | **수익모델 부재**: 어떻게 돈 버는지(수수료/SaaS/금융 take-rate) 화면·근거 없음 |
| 위치기반 일자리·경력카드·매칭 알림 = 현장 친화 UX, 사회적 가치 명확 | **GTM/공급 해법 공백**: 진짜 일용직 공고 확보 방법 미해결(워크넷 개인불가 등) |

→ 데모·내러티브·사회가치로 **본선권엔 들지만 "우승"엔 한 방이 부족**. 작은 파일럿 실데이터 + 기업 LOI 1~2건이면 판도가 바뀜.

## 2. VC 투자 — 아직 아님 (≈45/100)

- VC는 데모가 아니라 **트랙션·시장·팀**에 투자. 실사용자·실기업·리텐션 숫자가 없어 **텀시트 단계 아님**.
- 긍정: "이 팀은 빠르게 잘 만든다"는 강한 신호 + 측정 인프라 완비 → **웜 인트로·"X 보여주고 다시 오라"** 수준엔 충분.
- 관건: 실가입/재방문 코호트, 기업 수요 증거, 단위경제(unit economics) 가설.

## 3. SI(전략적투자자 — 건설대기업·인력공급사) — 부분 (≈55/100, PoC 가능)

- SI는 트랙션보다 **전략 적합성 + PoC + 데이터 자산**을 본다. 출역·경력·coworkers 데이터와 '출역부·정산 디지털화'는 협력사 대상으로 매력적.
- 관건: **PoC/MOU 1~2건**(건설사·인력사무소) + **직업안정법(유료 직업소개) 포지셔닝**("소개"가 아닌 "운영툴+게시판"). 없으면 "흥미롭지만 보류".

---

## 4. 구현됨 vs 부족함

- **구현됨(필요 기능 대부분 존재)**: 진입 로그인, 90초 프로필, 경력카드, 위치기반 일자리(지도+상세), 출역·정산, 관심기능 6종(Fake-door 검증), 매칭 알림(인앱+웹푸시), 기업 협약·공고·지원자 배정, 관리자 승인·현황, 마케팅 분석(퍼널·리텐션·Aha·관심), 이벤트 로그, demand-first 루프.
- **부족함(기능이 아니라 *증거/공급/수익*)**: 실사용자·실기업 트랙션, 진짜 일용직 공급원, 수익모델 구체화, 데이터 해자 입증, 규제(직업안정법) 정리, PoC/LOI 파이프라인 실적, 투자자용 데이터 대시보드.

## 5. 지금 더해야 할 우선순위 (영향순)

| 순위 | 액션 | 대상 | 영향 |
|---|---|---|---|
| 1 | **실검증 신호 확보** — 컨시어지로라도 실제 기업 공고 몇 건 + 실제 기술자 가입 + 작은 리텐션 숫자. 숫자 > 데모 | 대회·VC·SI | 최상 |
| 2 | **공급 한 채널 가동** — job-sourcing-plan의 ①출역 역류 / 반장 1줄 호출 / 인력사무소 1곳 파일럿 중 하나로 진짜 공고 흐르게 | 대회·SI | 최상 |
| 3 | **수익모델 명문화 + 화면화** — 매칭 수수료/구독/금융 take-rate를 IR·전략페이지에 구체화 | VC·SI | 상 |
| 4 | **데이터 해자 내러티브** — coworkers·출역 데이터가 왜 복제 불가 자산인지 슬라이드 + 초기 데이터 | VC·SI·대회 | 상 |
| 5 | **규제 정리** — 직업안정법 리스크 대응(운영툴+게시판 포지셔닝) | SI·VC | 상 |
| 6 | **PoC/LOI 파이프라인 가동** — /amono PoC 메뉴를 실제 LOI 1~2건으로 | SI·대회 | 중 |
| 7 | **투자자용 데이터 대시보드(Phase 5)** — 분석웹을 투자자 향(向) 뷰로 | VC·SI | 중 |

## 6. 5-렌즈 정밀 평가 (워크플로우 결과)

> VC·모두의창업·SI·제품갭·산업 5개 렌즈가 문서+코드를 정독해 평가. (종합 에이전트는 서버 과부하로 실패 → 5개 렌즈 결과를 직접 종합.)

**렌즈별 판정** (전부 `partial`): VC **52** · 모두의창업 **62** · SI **52** · 제품갭 **62** · 산업 **52** (평균 ≈56). → **종합: 부분 준비.** v1 진단과 일치.

### 6-1. 코드로 확인된 추가·심화 갭 (v1보다 날카로움)

- 🔴 **해자가 "주장"일 뿐 코드엔 없음**: `CareerCard.coworkers`는 자유텍스트 String 한 칸이라 "누가 누구와 일했나" 쿼리 불가. 매칭(`notifications.service`)은 직군∩·지역∩·경력≥ **단순 규칙**일 뿐 coworkers/출역 그래프를 안 씀. → VC/SI가 코드 보면 "서사는 그래프인데 실제는 룰"임을 간파. *(vc·si·industry·product-gap 공통)*
- 🔴 **일자리 첫 화면이 하드코딩 목업**: 홈/지도 탭은 '힐스테이트 송도·대주건설' 등 mock(MonoApp.tsx L633·639·829), 실 API는 '채용공고' 탭만. 심사위원이 클릭 몇 번에 "눈속임" 간파 시 신뢰 붕괴. *(product-gap)*
- 🔴 **출역 데이터가 감사 불가**: GPS/지오펜스·본인확인·전자서명·임금원장 없음, `workDate` 자유문자열. SI가 투자하는 단 하나의 이유(검증된 출역=노무·정산 근거)가 성립 안 함. *(si)*
- 🟠 **CLAUDE.md 규칙 #1 위반(즉시 수정 가능)**: 사용자 노출 모달에 '준비 중'·'협의 중' 문구 잔존(MonoApp.tsx L654/656/661 → modalStatus). 자기 규칙(미노출·관심신청형) 위반. *(product-gap)*
- 🟠 **전략(IR) 페이지 미구현**: `strategy-page-plan.md`에 Hero·BM·Deep Dive 7모달 설계됐으나 `web/app`에 strategy/deepdive 라우트 전무. 발표·VC 1차의 핵심 무기 부재. *(competition)*
- 🟠 **TAM/시장 사이징 없음**: 전략문서에 기업가치 사다리(10억~1조)는 있으나 건설 일용직 시장규모(공제회 가입자·일당총액·인력사무소 시장·중개마진) 근거 없음 → 'wishful'로 읽힘. *(vc)*
- 🟠 **팀(founder-market-fit) 정보 0**: 시드 투자의 70%는 팀, 특히 오프라인·규제헤비 시장은 "왜 당신이 새벽 인력시장에 닿나"가 결정적인데 증거 없음. *(vc·competition)*
- 🟠 **PoC/MOU/LOI 파이프라인 모델·실적 없음**: Phase3 P0인데 schema에 Interview/Partner/PoC 모델 없고 실적 0. SI는 'PoC→MOU→투자' 파이프로 움직임. *(si·product-gap)*
- 🟡 **수익모델·과금 트랜잭션이 데모에 없음** + **직업안정법 정합 미설계**(소개 vs 운영툴 격벽이 문서 플래그 수준). *(vc·competition·si·industry)*
- 🟡 **과확장 focus 리스크**: IR 장기비전에 'AGI Core OS Device·휴먼로봇·3D프린터'까지 → 초기투자엔 과대포장 신호, 단일 wedge로 좁혀야. *(vc)*
- 🟡 **검증 지표가 전부 목표치(target), 실측(actual) 0** · **사람인/경기데이터드림 베이스라인 어댑터도 미구현**(빈 일자리 탭). *(전 렌즈)*

### 6-2. 일치하는 강점 (5렌즈 공통 인정)
Demand-first 루프가 **슬라이드가 아니라 코드로 닫혀 있음** · 분석 대시보드가 **실연산**(mock 아님) · 규제 기능을 **관심신청 Fake-door**로 정직 처리 · job-sourcing-plan의 **산업 이해도** · 운영 수준 **UX 완성도**. → "이 팀은 빠르게 잘 만든다"는 강한 실행력 신호.

### 6-3. 종합 우선순위 (영향순, 5렌즈 합의)
1. **실검증 트랙션 0→1** — 컨시어지 파일럿(1지역·1~2직군: 기술자 30~50, 반장 3~5, 기업/사무소 2~3) → `/analys`를 진짜 숫자로.
2. **진짜 공급 1채널 코드화** — `JobSource`/externalId 스키마 + 출역 역류 또는 반장 1줄 호출.
3. **해자를 코드로 실현** — coworkers를 관계(엣지)로 정규화 + 출역 기반 "같이 일한 사람 재호출" 1경로 구현.
4. **수익모델 1개 구체화 + 직업안정법 정합** — SaaS/열람권 과금 + 비현금 포인트, 법무 의견서 1장.
5. **IR 자산** — 시장 사이징(TAM/SAM/SOM) + 팀 founder-market-fit + 전략(IR) 페이지 7모달 구현.
6. **PoC/MOU 1건 + 파이프라인 보드**(amono).
7. **🔧 즉시 수정 2건**: ① 홈/지도 하드코딩 목업 → 실데이터 바인딩, ② '준비 중/협의 중' 문구 제거(규칙 #1 위반).

### 6-4. 냉정한 결론
**필요 *기능*의 대부분은 구현돼 있다. 빠진 건 기능이 아니라 그 위에 쌓일 *증거*(실트랙션·공급·해자 실현·수익·PoC)다.** 모두의창업은 본선/입상권이나 우승엔 실데이터 한 줄이 결정타. VC는 아직 watchlist(텀시트 전), SI는 PoC 제안 단계(투자 직전 아님). **추가 개발보다 "작은 진짜 0→1"이 가장 큰 레버.**
