# 비활성화 / 비노출 기능 목록

사용자 앱(`web/app/mono`, 온보딩 등)에서 **아직 사용자에게 노출하지 않는** 기능을 정리합니다.
미구현·미연동이거나 규제·정책 검토가 필요한 항목으로, 대부분 **주석 처리**로 숨겨 두었고 추후 그대로 복원할 수 있습니다.

> 규칙: 사용자 노출 화면에 "준비 중/Beta/Prototype" 등의 문구를 직접 쓰지 않고, **숨김(주석) 처리**하거나 "관심 신청" UX로 전환합니다. (CLAUDE.md · `docs/user-app-guidelines.md`)

## 관심 신청으로 대체된 기능 (숨김 아님)

규제·미구현 기능 중 일부는 숨기는 대신 **"관심 신청/관심 등록"** UX로 노출합니다. (`내 정보 → 관심 기능 신청`)

- 경력 인증 · 금융 혜택 · 공구·장비 대여 · 체류·고용 관리 · 안심 정산 · 기업 열람권 (6종, `web/lib/constants.ts` `INTEREST_FEATURES`)

## 공개 프로필 공유(`/p/:id`) — 알려진 제약

프로필 공유 링크(`#3`)는 현재 다음 제약을 가집니다. 제품·보안 관점에서 인지/합의가 필요합니다.

- **식별자**: 공유 URL이 내부 기본키 `User.id`(cuid)를 그대로 사용합니다(`/p/{User.id}`). cuid라 순차 열거 위험은 낮지만, 같은 `id`가 `POST /users/:id/*` 등 다른(현재 비인증) 엔드포인트의 키와 동일합니다.
- **만료·철회 부재**: 링크는 만료가 없고, 한번 유출되면 철회 수단이 없습니다(영구 capability).
- **공개 범위 제어 부재**: "공개 범위 설정"이 미구현이라, 공유 시 경력·자격증·교육 이력 전체가 무조건 공개됩니다(아래 비공개 필드 제외).
- **비공개 필드**: 휴대폰·이메일·자격증 발급번호(`licenseNo`), 경력카드의 `coworkers`(제3자 실명)·`memo`(사적 메모)는 서버 `select` 화이트리스트로 응답에서 제외합니다(`api/.../users.service.ts getPublicProfile`).
- **권장(중기)**: 내부 PK 대신 회전·비활성화 가능한 별도 `shareToken`(랜덤, nullable)을 도입해 `/p/{shareToken}` 으로 조회하도록 분리.

## 복원 방법

각 항목은 해당 위치의 주석(`{/* ... */}` 또는 줄 주석)을 해제하면 다시 노출됩니다.
복원 시 연결된 동작(핸들러·라우팅·데이터)이 실제로 구현되었는지 먼저 확인하세요.
