# 채용공고 소싱 방안 (Job Sourcing Plan)

> 작성: 2026-06-23 · 상태: **방안 검토(미구현)** · 목적: MONO 일자리 탭에 **실제 건설 채용공고**를 지속 공급하는 방법 총정리
>
> 현재 공고 소스는 파트너웹 **수동 입력**이 유일. 이 문서는 "외부의 실제 진행 중 공고를 가져오는(런칭 운영)" 방향을, **API 수집(베이스라인)** 과 **비-API 창의 방안(코어)** 으로 나눠 정리한다.

---

## 0. 목표와 결정된 방향

- **1차 목적**: 외부의 실제 건설/현장 채용공고를 가져와 **MONO 앱 안에서 네이티브로 파싱·열람**. (데모 시드/직접등록 유도가 아님)
- **결정 사항**:
  1. 가져온 공고는 외부 redirect가 아니라 **MONO 카드 + 상세 시트(기구현)로 네이티브 열람**.
  2. **지원(apply) 기능은 지금 구현하지 않음** — 열람까지가 1차 범위.

---

## 1. 핵심 통찰 — MONO의 불공정 우위 (왜 API만으론 안 되나)

> **API 수집은 누구나 하는 commodity이고, 진짜 일용직(새벽 인력시장 '가다')엔 닿지도 못한다.** 일용직은 인력사무소·카카오 단톡/오픈채팅·문자·반장 구두로 **오프라인 유통**된다.

MONO만 가진 두 자산이 진짜 무기다:

- **반장–팀 `coworkers` 그래프** — 누가 누구랑 같이 일했나 (CareerCard.coworkers).
- **실제 출역(`Attendance`) 데이터** — 누가 오늘 진짜 나왔나.

→ 문제를 "외부에서 긁어오기"가 아니라 **(a) 현장 한복판의 우리 사용자(반장·기능공)가 흘려보내게 하고 (b) 이미 가진 데이터로 공급을 재생산**하는 것으로 재정의한다. 이게 경쟁사가 복제 못 하는 해자다.

## 2. 비-API 공급 방안 (코어) — 가장 강한 5수단

| | 수단 | 메커니즘 한 줄 | 난이도 | 일용직 도달 |
|---|---|---|---|---|
| ① | **출역 역류 루프** | 체크아웃 때 "오늘 나온 사람 내일 또?" 원탭 → 익일 재호출. **기존 데이터만으로 공급 자동 재생산** | 🟢 낮음 | ✅ |
| ② | **단톡/문자 1탭 인입** | 카톡·SMS·전단지 모집글을 공유/붙여넣기/사진(OCR) → LLM 공고화 + 같은 글 N명 교차검증 dedup | 🟡 중 | ✅ |
| ③ | **반장 1줄 호출** | "내일 평택 갱폼 5명 일당22 새벽5시" 복붙/음성 → LLM 슬롯필링 → 같이 일한 인부에 즉시 푸시(콜드스타트 0) | 🟡 중 | ✅ |
| ④ | **역방향 가용 보드** | 노동자가 "나 내일 빈다(직군·지역·일당)" 등록 → 반장·사무소가 일감 들고 들어옴. **공급 0에서도 도는 single-player** | 🟡 중 | 부분 |
| ⑤ | **인력사무소 무료 배차 SaaS** | 채용플랫폼 아닌 '새벽 배차·정산' 무료툴 → 업무 부산물로 데이터, **결원만** 공개(탈중개 방어) | 🔴 높음 | ✅ |

### ① 출역 역류 루프 — "오늘 나온 사람 내일 또?" *(난이도 낮음 · 일용직 ✅)*
- **메커니즘**: 체크아웃(Attendance) 순간 반장 화면에 '오늘 나온 N명 중 내일도 부를 사람' 원탭 다중선택 → '○○현장 내일 06:30 재출역 제안' 푸시 → 수락 시 익일 JobApplication 자동 생성. 연속 출역자는 '단골 명단'으로 자동 우선 호출. 경력카드 기반 '어제 그 현장' 재모집 프리필(복제+수정 30초).
- **왜 최적**: 일용직은 매일 새로 구하는 게 아니라 "쓸 만한 사람을 계속 다시 부르는" 구조. MONO만 가진 '실제 출근' 데이터로 공급을 **획득이 아닌 재생산**으로 바꾼다. 가장 가볍고 콜드스타트 이후 지속성을 만든다.
- **첫 발**: Attendance 체크아웃 이벤트에 '내일도 부를 사람' 원탭 → 익일 재출역 제안 푸시 → 수락 시 JobApplication 자동생성. 반장에게 출역부 자동생성 효용 함께 노출.

### ② 단톡/문자 1탭 인입 (공유시트·붙여넣기·OCR) + 교차검증 dedup *(난이도 중 · 일용직 ✅)*
- **메커니즘**: 안드로이드 Web Share Target + 만능 붙여넣기창 + 사진 OCR로 카톡·문자·벽보 원문 인입 → LLM이 JobPost로 구조화 → PENDING(기존 관리자 승인 라인 재사용). 원문 해시+현장명+날짜로 dedup, 같은 글 N회 제보 시 "3명이 같은 공고를 봤어요" 신뢰가중. 연락처 자동 마스킹. 원 게시자에게 "내 공고가 N명에게 닿았다" 회신링크 → 공급자로 역유입.
- **왜 최적**: 사용자(반장·기능공)가 바로 그 오픈채팅·구인 단톡 한복판에 물리적으로 존재. "평소 보던 카톡을 공유 한 번"이라 마찰 최저, 붙여넣기는 iOS·고령층까지 닿는 최저공통분모 폴백. 공유/이벤트로그 인프라 이미 보유.
- **첫 발**: 홈에 '받은 일자리 붙여넣기' 단일 입력창 + LLM 공고판별/구조화 + 해시 dedup(가장 가벼운 폴백) → 이후 Share Target·사진 OCR 추가.

### ③ 반장 1줄 호출 (복붙·음성·1탭) → coworkers 자동 푸시 *(난이도 중 · 일용직 ✅)*
- **메커니즘**: 반장이 "내일 평택 갱폼 5명 일당22 새벽5시 평택역"을 붙여넣거나 음성 1줄 → LLM이 직군·인원·일당·지역·집결시간 슬롯필링 → JobPost(source=FOREMAN, oneOff, 즉시 OPEN) 초안. '확인' 1탭으로 (1)과거 Attendance·coworkers 연결 인부에게 푸시, (2)카톡에 뿌릴 공유링크 자동 생성. 인부 '나 감' 1탭 → JobApplication → Attendance 체인. 차면 '마감' 토글 + 출역부 자동 생성.
- **왜 최적**: 일용직 1차 유통 노드(반장)가 이미 사용자로 들어와 있고, coworkers·Attendance 그래프 덕에 첫 푸시 대상이 0에서 시작하지 않음. 반장은 "늘 쓰던 문장 복붙"만 하면 됨. 이 한 줄이 곧 API 어디에도 없는 진짜 당일 가다.
- **첫 발**: JobPost에 source=FOREMAN·oneOff·rawText 필드 추가 + 모집 문장 1줄 → 11직군·7지역버킷·일당·집결시간 LLM 슬롯필링 + "이 내용 맞아요?" 확인칩 화면.

### ④ 역방향 가용 보드 — 노동자가 "나 내일 빈다" *(난이도 중 · 일용직 부분)*
- **메커니즘**: 노동자가 출역/경력카드 쓰는 김에 '가용 등록' 옵트인(직군·지역버킷·날짜·희망일당) → 검색 가능한 '가용 인력 보드'. 반장·사무소·운영자가 "내일 서울 철근 3명, 경력 5년+" 필터 → '호출' 1탭 → 비공식 공고 즉석 매칭. 초기엔 운영팀이 보드 보고 직접 매칭(컨시어지 결합).
- **왜 최적**: 양면 중 노동자는 이미 보유 → 공고 0이어도 노동자 자산을 가용 풀로 전환하면 닭-달걀을 **한쪽만으로 점화**. CareerCard·직군·지역이 그대로 필터 축. 콜드스타트 정공법.
- **첫 발**: 경력카드/출역 화면에 옵트인 '가용 등록' 토글 + amono에 가용 인력 필터 보드 + '호출' 버튼(초기 수동 매칭).

### ⑤ 인력사무소 무료 배차·정산 SaaS — 결원만 자동 공개 *(난이도 높음 · 일용직 ✅)*
- **메커니즘**: 사무소에 채용플랫폼이 아니라 매일 새벽 어차피 하는 **배차·출역·일당정산 무료 SaaS** 제공. 소장이 "내일 평택 형틀 8명 일당18" 한 줄 입력 → 자기 인력 배정·연락·출결·정산. 이 입력이 곧 JobPost(companyId nullable, source=AGENCY). **자기 인력으로 못 채운 '결원'만** 토글로 MONO 풀에 공개. "인력풀은 사무소 소유, 결원만 공개"를 UI·계약으로 격벽 → 탈중개 공포 차단. '소개'가 아닌 '운영툴+게시판' 포지셔닝으로 직업안정법 규제선 회피.
- **왜 최적**: 새벽 가다는 어떤 API에도 없고 오직 사무소가 쥐고 있음. 사무소 1곳 장악 = 매인 수십~수백 노동자의 일용직 흐름 전체 확보. MONO의 Attendance·정산 도메인이 출역부 앱으로 자연 확장. **진짜 공급에 닿는 가장 두꺼운 파이프.**
- **첫 발**: 좁은 1지역·직군에서 사무소 3~5곳 컨시어지 온보딩. 먼저 amono에 '빠른 공급 입력(30초 정형화)'으로 운영자가 손수 결원 발행 → 가치 증명 후 셀프 배차툴로 확장.

### 비-API 발상 지도 — 12 테마 전체

1. **반장/십장 발신 공고 — 구두 모집을 그대로 삼키기**: 카톡·구두 모집 문장을 새 폼 없이(복붙·음성·1탭) 받아 비공식 JobPost로, coworkers·Attendance로 즉시 푸시. → ③의 모태.
2. **단톡·오픈채팅·문자 인입 — 비공식 채널 크라우드소싱 캡처**: 공유시트/붙여넣기/사진/포워딩 1탭 인입, 해시 dedup+교차검증. → ②의 모태.
3. **출역 데이터 역류 — 끝난 일을 다음 공고로 되먹임**: 체크아웃을 다음날 수요로. → ①의 모태.
4. **역방향 마켓 — 노동자 가용성을 미끼로 수요를 끌어오기**: "나 내일 빈다" 보드. → ④의 모태.
5. **평판·신뢰 원장을 미끼로 수요측 역유입**: 반장·사무소의 '노쇼' 고통을 출역 기반 평판 원장(약속 지킴 지수)으로 무료 해결 → 상주시킨 뒤 공고로 전환. MONO만 '실제 출근'을 안다.
6. **인력사무소 트로이목마 — 무료 운영툴로 결원만 흡수**: → ⑤의 모태.
7. **팀(크루) 단위 공급 — 반장이 팀을 통째로**: 한국 현장 단위는 개인이 아니라 '반장+팀 5~10명'. 검증된 '팀 카드' 매물 등록 / 크루 초대로 반장당 5~10명 한 번에 유입. 팀 도급 수요와 직결, 반장 1명 초대 레버리지.
8. **컨시어지 + 초집중 점화 — 안 확장되는 걸 손으로**: 한 현장/지역·직군으로 좁혀 운영자가 너저분한 카톡을 손으로 30초 정형화 발행 / 단일 대형현장 게이트·함바 QR로 닫힌 그래프 안 작은 루프 완결.
9. **오프라인 거점 QR 브릿지 — 종이 공고를 디지털로**: 인력시장 게시판·사무소 유리창·자재상·함바 벽보에 QR 배포 → 사진+한 줄 인입. 사용자 밀집지가 곧 영업 지도.
10. **보상형 크라우드소싱 — 제보·추천에 출역검증 보상**: '소개비(데마찌)' 문화를 합법·소액·투명 버전으로. **등록이 아닌 '출역 발생' 시점 보상**으로 어뷰징 차단. ⚠️ 직업안정법 리스크 최상위 → 비현금 포인트·법무 선행.
11. **상위 조직/원청 톱다운 제휴 — 한 번에 수백 곳**: 전문건설협회·직종조합 MOU, 시공사 협력사 포털, 의무지출 서비스(안전·노무대행·건강검진) 물물교환, 공공 인력은행(공제회·지자체·노조). 레버리지 큼, 의사결정 느림, 진짜 가다 도달은 부분적.
12. **선행지표 — 자재·장비 발주로 수요 예측**: 자재·장비 발주 증가 → 며칠 뒤 인력 수요. MONO Gear(장비대여) 데이터로 '곧 사람 필요할 현장' 선행 포착·선점 넛지. 가장 투기적.

## 3. 베이스라인 — 공개 API 수집 (commodity tier)

비공식 채널이 코어지만, '정식·반정식 건설 채용'을 싸게 까는 베이스라인으로 병행 가치 있음.

### 제약 (검증됨)
- **워크넷/고용24 '채용정보목록·상세' API는 개인 신청자에게 안 열림.** 개인은 채용행사(2025-02-11부터)·공채속보·공채기업정보만. → 전국 워크넷 풀데이터는 **MONO 사업자등록 후** 가능.

## 4. 공통 아키텍처 — Source-Pluggable Ingest

API든 비-API든 입력 경로만 다르고 **공통 파이프라인 + 소스별 어댑터**로 흡수한다.

```
[입력원] ──▶ [SourceAdapter]  ──정규화──▶  JobPost(source, externalId, …)
 API(사람인…)    fetch/parse                 │  upsert = 중복/갱신 안전
 반장 1줄/붙여넣기  LLM 슬롯필링·구조화           ▼
 출역 역류         Attendance→재수요         /mono 일자리 탭(기존 카드/상세 재사용)
 사무소 SaaS       결원 토글 공개
```

### 데이터 모델 변경 (`api/prisma/schema.prisma`)
- `source` enum `JobSource { MONO | FOREMAN | INBOUND | AGENCY | SARAMIN | WORKNET … }` (기본 `MONO`)
- `externalId String?` + `@@unique([source, externalId])` — 중복방지 upsert 키(외부 공고·제보 dedup)
- `externalUrl String?`, `sourceName String?`(회사/사무소/반장명), `rawText String?`(원문 보존·재파싱)
- `companyId String?` **nullable**(외부·반장·사무소 공고는 MONO 기업계정 없음)
- (선택) `oneOff Boolean`(당일/단발), `salary`, `meetAt`(집결시간·장소), `deadline`
- LLM 매핑: 비정형 텍스트/외부코드 → 11직군(`JOB_TYPES`)·7지역버킷(`REGIONS`)·`CareerBand`

### 노출
- 기존 일자리 카드/상세 시트 재사용. 소스 배지(반장/사무소/사람인/워크넷). 외부·공개 출처는 **출처표시** 준수. `source !== MONO`는 지원 버튼 보류(미구현).

## 5. 권장 시퀀스

1. **지금(가장 싸고 우리 데이터만으로)**: ① 출역 역류 + ② 붙여넣기 인입(가장 가벼운 폴백부터)
2. **점화 엔진**: ③ 반장 1줄 호출로 반장 발신 공고 흡수
3. **초기 공백 메우기**: ④ 역방향 보드 + 운영자 수동 매칭(컨시어지)
4. **두꺼운 파이프**: ⑤ 인력사무소 SaaS는 한 지역·직군에서 가치 증명 후 확장
5. **베이스라인 병행(선택)**: 사람인/경기데이터드림 어댑터로 정식 건설 채용 보충

## 6. ⚠️ 법적 플래그 — 직업안정법

- **보상형 크라우드소싱·소개 커미션·휴먼 어그리게이터**(테마 10, b2b '공급마스터')는 **유료 직업소개 규제(직업안정법)** 해석 리스크 **최상위**.
- 시작은 **비현금 포인트** + **노무·법무 검토 선행**. 인력사무소 SaaS(⑤)도 '소개'가 아닌 '운영툴+게시판' 포지셔닝으로 규제선 회피 설계 필요.

## 7. 의존성 / 미결정

- **API 키(사용자만, .env 커밋 금지)**: 사람인 <https://oapi.saramin.co.kr/join> → `SARAMIN_API_KEY` / 경기데이터드림 → `GG_DATA_KEY` / (이후 사업자)워크넷.
- **LLM 파싱 비용·정확도**: ② ③에 LLM 슬롯필링 필요 → 모델·프롬프트·검증 UX(확인칩) 설계.
- **미결정**:
  1. 첫 프로토타입 집중 수단(① vs ② vs ③ vs ④/컨시어지)
  2. 가져온 공고 자동 OPEN vs 관리자 검수(PENDING) — 외부 API는 자동 OPEN, 제보/비정형은 PENDING 제안
  3. 점화 지역·직군(한 게이트/지역 집중 여부)
  4. 보상·인센티브 도입 시점(법무 게이트)
  5. 지원(apply) 연결 시점
