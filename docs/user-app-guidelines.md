# MONO 사용자 앱 개발 업무 고려사항

> 출처: 내부 기획 문서 "MONO 사용자 앱 개발 업무 고려사항 문서" (정식 오픈형 MVP 개발 원칙)
> 이 문서는 사용자 앱 개발 시 **반드시 준수**해야 하는 기준입니다.

## 1. 문서 목적

MONO 사용자 앱은 단순 프로토타입이 아니라 **정식 오픈을 전제로 한 MVP**입니다.
초기 앱에서도 실제 서비스처럼 신뢰하고 사용할 수 있는 **완성도, 데이터 수집 구조,
마케팅 분석 구조, 규제 대응형 UX, 사용자 친화적 문구**를 함께 고려합니다.

## 2. 제품 개발 기본 원칙

### 2-1. 정식 오픈형 MVP

| 항목 | 개발 기준 |
| --- | --- |
| 화면 완성도 | 실제 서비스처럼 보이는 UI/UX |
| 기능 문구 | 사용자에게 제공 가능한 기능 중심으로 표기 |
| 데이터 수집 | 가입, 프로필 작성, 경력 등록, 공유, 재방문 행동 추적 |
| 규제 대응 | 민감 기능은 안내형 UX로 처리 |
| 테스트 방식 | PC 웹, 모바일 웹, 앱 화면 모두에서 검증 가능 |
| 배포 단계 | iOS/AOS 테스트 앱 배포 **전 단계**까지 준비 |

### 2-2. 문구(Copy) 원칙 — 노출 금지 표현

사용자가 혼란을 느낄 수 있는 개발용/내부 용어/미확정 기능 표현을 **노출하지 않습니다.**

| 지양 표현 | 이유 |
| --- | --- |
| Beta, Prototype, Test | 정식 서비스 신뢰도 저하 |
| 준비 중입니다 | 사용자 가치 전달 약화 |
| 개발 예정 기능 | 현재 가능한 기능으로 오해 |
| 내부 검증용 | 서비스 완성도 저하 |
| Fake Door | 사용자 노출 부적절 |
| North Star, Retention, Aha Moment | 내부 분석 용어 |

### 2-3. Fake Door → 사용자 표현 매핑

| 내부 의미 | 사용자 표현 |
| --- | --- |
| 프로필 검증 테스트 | 내 경력 인증 관심 신청 |
| 금융 수요 테스트 | 금융 혜택 관심 등록 |
| 기업 조회 수요 | 기업에게 보여줄 프로필 만들기 |
| 외국인 비자 관리 테스트 | 체류·고용 관리 관심 등록 |
| 공구·장비 쉐어 테스트 | 공구·장비 대여 관심 등록 |

## 3. 사용자 친화적 디자인 원칙

토스, 당근처럼 **단순하고 직관적인** 사용 경험을 지향합니다.
대상 사용자: 산업 현장 근로자, 기술자, 중장년층, 외국인 근로자, 경력전환자.

| 원칙 | 설명 |
| --- | --- |
| 한 화면 한 행동 | 한 화면에서 해야 할 행동을 하나로 명확화 |
| 짧은 문장 | 긴 설명보다 핵심 문장 중심 |
| 쉬운 단어 | 스타트업·금융·데이터 용어보다 생활 언어 |
| 진행률 표시 | 프로필 완성도, 등록 단계, 다음 행동 안내 |
| 즉시 보상 | 입력 후 바로 경력 카드, 미리보기, 공유 링크 제공 |
| 신뢰감 | 개인정보, 공개 범위, 기업 열람 여부 명확히 안내 |
| 모바일 우선 | 현장에서 손쉽게 입력 가능한 모바일 UX |

## 4. 핵심 시나리오

### 4-1. 가입 (최대한 짧게)
휴대폰/이메일 가입 → 기본 정보 → 직군 선택 → 경력 연차 → 희망 지역 → 프로필 생성 완료

### 4-2. 프로필 작성 (가치 체감)
90초 기본 프로필 → 경력 카드 1건 → 자격증/교육 → 완성도 확인 → 기업 미리보기 → 공유 링크 생성

### 4-3. 이용 (재방문 유도)
경력 추가 / 프로필 수정 / 공유 / 관심 기능 신청 / 기업 조회 알림 / 공구·장비 관심 등록

## 5. 규제·API 연동 필요 기능 — "관심 신청" 처리

다음 기능은 규제 검토·제휴·공공/기업 API가 필요하므로 **관심 신청/안내 화면**으로 제공하고,
클릭 시 사용자 친화적 안내 문구를 노출합니다.

| 기능 | 필요한 검토 |
| --- | --- |
| 경력 인증 | 기관·기업 확인, 경력 증명 방식 |
| 안심 정산 | 전자금융, 지급대행, PG 제휴 |
| 보험 연계 | 보험사 제휴, 상품 구조 |
| 금융 혜택 | 금융기관 제휴, 동의 구조 |
| 자격증 자동 연동 | 공공 API, 자격 데이터 연동 |
| 외국인 체류·고용 관리 | 법무·노무·출입국 검토 |
| 공구·장비 대여 | 대여 계약, 보증금, 보험, 파손 책임 |
| 중장비 연결 | 운전 자격, 안전교육, 장비 보험 |

> 안내 문구 톤 예시: "경력 인증 기능은 더 신뢰도 높은 프로필을 만들기 위한 기능입니다.
> 현재 관심 신청을 받고 있으며, 신청해주신 분께 우선 안내드리겠습니다."

## 6. 마케팅 분석 — 필수 이벤트 로그

가입/프로필/경력/관심/공유/재방문 단계에서 이벤트 로그를 수집합니다.

**가입**: `page_view`, `signup_started`, `signup_completed`, `onboarding_completed`,
`job_type_selected`, `region_selected`, `career_year_selected`

**프로필**: `profile_started`, `profile_basic_completed`, `career_added`,
`career_three_added`, `certificate_added`, `education_added`, `profile_completed`,
`profile_previewed`, `profile_shared`, `pdf_downloaded`

**관심 기능**: `career_verification_interest_clicked`, `finance_benefit_interest_clicked`,
`equipment_rental_interest_clicked`, `foreign_worker_management_interest_clicked`,
`safe_payment_interest_clicked`, `company_view_interest_clicked`

**재방문**: `return_visit`, `profile_updated`, `career_updated`, `notification_clicked`

## 8. 테스트·배포 원칙

현 단계 목표: **iOS/AOS 테스트 앱 배포 전 단계**까지. 모든 기능은 PC 웹·앱 화면·모바일 웹에서 테스트 가능.

검증 항목: 반응형(모바일·태블릿·PC), 이벤트 로그 정확 수집, 관심 기능 안내 문구,
개인정보/프로필/행동 데이터 분리 저장.

## 9. 우승 필수 기능

### 9-1. 사용자 앱 (전부 P0)
회원가입 · 90초 기본 프로필 · 직군 선택 · 경력 연차 · 경력 카드 · 자격증/교육 ·
프로필 완성도 · 기업용 미리보기 · 프로필 공유 링크 · 관심 기능 신청 · 이벤트 로그 · 내부 분석 대시보드

### 9-2. 기업용 최소 기능
- **P0**: 기업 관심 신청, 기술자 프로필 샘플 조회, 관심 기술자 저장
- **P1**: 인력 요청 양식, PoC 관심 신청, 장비 요청 관심 신청

## 10. 검증 가설 (요약)

- **기술자**: 경력/자격 프로필화 의향, 경력 1건+ 등록, 경력 3건+ 그룹의 높은 재방문, 자격/교육이 신뢰에 기여, 공유의 실질 가치, 기업 조회 알림이 재방문 유발
- **기업**: 경력/자격/교육 열람 의향, 관심 기술자 저장, 현장 경험·안전교육 중시, PoC 관심, 장비+기술자 동시 요청
- **MONO Gear**: 공구·장비 대여 의향, 전문 시공 공구 수요, 장비+기술자 패키지, 보증금·보험 시 신뢰↑, 장비 보유자 등록 의향
- **외국인 인력**: 체류·고용 관리 관심, 안전교육 이력 관리 중요, 고용 가능 기간 알림, 다국어 프로필
- **금융·보험**: 근무이력 기반 금융 혜택, 보험 연계, 장비 파손·분실 보험, 기업의 안전·보험 데이터 열람 의향

## 11. 화면별 고려사항 (요약)

- **온보딩**: 첫 문장 "내 경력과 기술을 신뢰 프로필로 만들어보세요." / CTA "프로필 만들기" / 설명 3줄 이하 / `onboarding_viewed`, `onboarding_cta_clicked`
- **기본 프로필**: 이름·직군·경력연차·희망지역, 선택형 입력, 90초 목표 / `profile_started`, `profile_basic_completed`
- **경력 카드**: 현장명·작업분야·근무기간·역할 (+사용장비·동료·메모), 카드형 / `career_added`, `equipment_used_added`
- **프로필 완성도**: 0~100%, 다음 행동 유도, 보상=기업 미리보기 / `profile_completion_viewed`
- **관심 기능**: 신청형 카드, "관심 등록하면 먼저 안내드릴게요." / `interest_feature_clicked`, `interest_submitted`
